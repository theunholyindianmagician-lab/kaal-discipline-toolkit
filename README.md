## What this gives you

**Five reusable patterns.** Pick the ones that apply.

### 1. Tier-S / Tier-W discipline template
- **Tier-S** = falsifiable, must wear derivation tag, machine-verifiable at the smallest scale where it manifests.
- **Tier-W** = postulated witness / cultural provenance, honored with explicit boundary, NEVER asserted as derivation.
- **Bridge** = Tier-S theorem + Tier-W naming, with explicit "what is NOT claimed" disclaimer.
- **See:** `templates/tier-discipline-template.md`

### 2. HALAHALA-MANIFEST template
- Every retraction in writing, timestamped, tagged with error-type.
- The manifest grows in the same commits as new claims (audits scale as theorems).
- Pre-seeded error patterns: encoding-tautology, post-hoc factorization, cross-adic category error, GPS-coordinate-residue, vocabulary-borrowing-as-derivation, instance-vs-theorem-conflation, milestone-rhetoric-inflation.
- **See:** `templates/halahala-manifest-template.md`

### 3. Pre-registration workflow
- Hash-commit prediction BEFORE submitting the test.
- Record the falsification criterion at the same moment.
- Report result publicly whether it passes or fails.
- **See:** `templates/preregistration-template.md`

### 4. 9-tag derivation checklist
A universal derivation taxonomy (transposable to any substrate):
- **(i)** foundational axioms (your substrate's algebraic / categorical / physical starting points)
- **(ii)** generator-status / canonical-choice justification
- **(iii)** period / cycle / orbit determination
- **(iv)** invariance-lock / preservation law
- **(v)** liberation / exit-exponent
- **(vi)** hardware / external-sensor grounding
- **(vii)** cultural / namespace tagging (provenance, not derivation)
- **(viii)** ideal / sub-substrate stratification
- **(ix)** restricted-domain analysis
- Every claim wears one or more tags. **Untagged = provisionally suspect.**
- **See:** `templates/derivation-tag-checklist.md`

### 5. Pañca-pramāṇa boundary checker
The five Nyāya means of valid knowledge, applied as epistemic-firewall:
1. **Pratyakṣa** (direct measurement / perception)
2. **Anumāna** (formal inference)
3. **Upamāna** (analogy / homological mapping)
4. **Śabda** (authoritative testimony / cited lineage)
5. **Arthāpatti** (postulation from circumstance / "what must have been wrong")

Every claim is sorted into one pramāṇa. Each pramāṇa has its own evidence-rules. **No cross-pramāṇa claim-promotion without explicit bridge-disclaimer.**
- **See:** `templates/panca-pramana-checker.md`

## The minimal toolkit (one file, ~200 lines)

If you want the absolute minimum, just adopt the **Tier-S / Tier-W split**:

```markdown
# CLAIM
<your claim>

# TIER
- [ ] Tier-S (algebraic / formal / machine-verifiable)
- [ ] Tier-W (cultural / authoritative-testimony / provenance)
- [ ] Bridge (S + W with explicit boundary)

# DERIVATION TAG (if Tier-S)
- [ ] (i) foundational-axiom
- [ ] (ii) generator-status
- [ ] (iii) period-determination
- [ ] (iv) invariance-lock
- [ ] (v) liberation / exit-exponent
- [ ] (vi) hardware-grounding
- [ ] (vii) namespace-provenance (Tier-W marker)
- [ ] (viii) sub-substrate-stratification
- [ ] (ix) restricted-domain-analysis

# WHAT IS NOT CLAIMED
<explicit list of things this claim does NOT entail>

# SMALLEST k AT WHICH VERIFIED
<state the smallest scale at which the claim has been demonstrated>
```

That's it. Five sections per claim. Discipline emerges.
