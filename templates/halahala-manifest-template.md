## ◈ Common error patterns (pre-seeded forbidden-list)

These patterns have been retracted in the originating KAAL framework. Add your own as you discover them.

1. **encoding-arithmetic tautology** — choose an encoding, run arithmetic, observe outputs, relabel outputs as substrate predictions.
2. **post-hoc factorization mysticism** — find a coincidence after the fact, assert it as predicted.
3. **cross-adic category error** — treat results at one adic structure as if they hold at another.
4. **GPS-coordinate ↔ residue claims** — assert that terrestrial coordinates encode substrate residues without a derivation.
5. **vocabulary-borrowing as derivation** — use physics or computer-science terminology in a context where you have not done the corresponding physics or computer science.
6. **instance-vs-theorem conflation** — count instance-test passes as theorem count.
7. **rhetoric-inflation by milestone-K** — name large-k milestones as if they carry epistemic weight when they only carry repetition.
8. **universal-threshold-disguised-as-chip-relative** — use one chip's calibration as a universal cutoff across chips.
9. **sample-size-uncalibrated thresholds** — set tight thresholds (e.g. 5%) that the sample size cannot statistically support.
10. **bipolar-pole-asymmetry-asserted-as-physics** — observe a hardware target-asymmetry and attribute it to substrate properties rather than chip-routing.

## ◈ Operational rule

When you retract a claim:

1. **File the entry IN THE SAME COMMIT as the result that forced the retraction.** No commit-separation between "discovery" and "retraction" — they belong together.
2. **Pattern-match against the forbidden-list above.** If your retraction fits an existing pattern, reference it. If not, add a new pattern.
3. **Audit prior claims for the same pattern.** Per the discipline: *audits scale as theorems* — a single retraction triggers reviewing all related claims for the same error.
